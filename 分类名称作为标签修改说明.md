# 分类名称作为标签修改说明

## 修改概述
将商品列表页面的数据获取逻辑从使用分类ID改为使用分类名称作为标签，这样当用户点击"橘子"分类时，系统会使用"橘子"作为标签来获取相关商品。

## 修改的文件

### 1. 分类页面
**文件：** `Client/pages/goods/category/index.js`

#### 修改内容：
- **传递分类名称** (第18-32行)：
  ```javascript
  onChange(e) {
    // 获取点击的分类信息
    const { item } = e.detail;
    // 获取分类名称作为标签
    const categoryName = item.name || '';
    
    console.log('[onChange] 分类点击:', {
      groupId: item.groupId,
      name: item.name,
      categoryName: categoryName
    });
    
    // 导航到商品列表页面并传递分类名称作为标签
    wx.navigateTo({
      url: `/pages/goods/list/index?tag=${encodeURIComponent(categoryName)}`,
    });
  },
  ```

### 2. 商品列表页面
**文件：** `Client/pages/goods/list/index.js`

#### 修改内容：

- **URL参数解码** (第243-260行)：
  ```javascript
  onLoad(options) {
    const { groupId = '', tag = '' } = options;
    // 优先使用 tag 参数，如果没有则使用 groupId
    let goodTag = tag || groupId || '';
    
    // 如果tag参数被URL编码了，需要解码
    if (tag) {
      try {
        goodTag = decodeURIComponent(tag);
      } catch (error) {
        console.error('[onLoad] URL解码失败:', error);
        goodTag = tag;
      }
    }
    
    console.log('[onLoad] 加载商品列表页面:', {
      originalTag: tag,
      originalGroupId: groupId,
      decodedTag: goodTag
    });
    
    this.setData({ groupId: goodTag }, () => {
      this.init(true);
    });

    // 使用节流包装handleFilterChange
    this.throttledFilterChange = throttle(this.handleFilterChange.bind(this), 300);
  },
  ```

- **日志信息更新** (第165行)：
  ```javascript
  console.log('[init] 开始根据标签获取商品列表, 标签:', groupId);
  
  // 使用分类名称作为标签来获取商品
  const goodsList = await getGoodsByTag(groupId, 0); // 使用默认用户ID 0
  ```

## 数据流向

### 修改前：
1. **分类点击** → 传递 `groupId` (如: "2101")
2. **商品列表页面** → 使用 `groupId` 作为参数
3. **API调用** → `getGoodsByTag("2101", userId)`

### 修改后：
1. **分类点击** → 传递分类名称 (如: "橘子")
2. **商品列表页面** → 解码并获取分类名称
3. **API调用** → `getGoodsByTag("橘子", userId)`

## 核心逻辑

### 分类页面逻辑：
- 获取点击的分类的 `name` 字段
- 使用 `encodeURIComponent` 对分类名称进行URL编码
- 通过 `tag` 参数传递给商品列表页面

### 商品列表页面逻辑：
- 优先使用 `tag` 参数
- 对URL编码的分类名称进行解码
- 使用解码后的分类名称作为标签调用API

### 安全措施：
- **URL编码**：防止特殊字符在URL中造成问题
- **错误处理**：URL解码失败时的容错处理
- **参数兼容**：保持对 `groupId` 参数的向后兼容

## 使用示例

### 分类点击流程：
1. 用户点击"橘子"分类
2. 系统获取分类名称："橘子"

4. 页面跳转：`/pages/goods/list/index?tag=橘子`
5. 商品列表页面解码：`decodeURIComponent("橘子")` = "橘子"

1. **语义化标签**：使用分类名称作为标签，更直观易懂

3. **错误容错**：URL解码失败时的容错处理
4. **向后兼容**：保持对原有 `groupId` 参数的支持
