# 库存逻辑修改说明

## 修改概述
将库存判断逻辑从原有的 `spuStockQuantity` 字段改为基于数据库中的 `repertory` 字段，用于判断商品是否售罄和最大可购买数量。

#### 修改内容：
- **库存状态判断** (第474-481行)：
  ```javascript

    isStock: hasStock,
  ```

- **数量变更控制** (第380-395行)：
  ```javascript
  changeNum(e) {
    const { buyNum } = e.detail;
    const { stockQuantity = 0 } = this.data;
    
    // 确保购买数量不超过库存
    const maxQuantity = Math.max(0, stockQuantity);
    const finalBuyNum = Math.min(buyNum, maxQuantity);
    
    this.setData({
      buyNum: finalBuyNum,
    });
  }
  ```

## 核心逻辑

### 库存判断条件：
1. **有库存**：`repertory > 0`
2. **售罄**：`repertory <= 0`
3. **最大可购买数量**：`repertory` 的值

### 数据流向：
1. **数据库** → `repertory` 字段存储库存数量
2. **API响应** → 包含 `repertory` 字段
3. **前端服务** → 计算 `isStock`、`soldout`、`maxPurchaseQuantity`
4. **UI组件** → 根据库存状态控制用户操作

### 兼容性处理：
- 保留原有的 `spuStockQuantity` 字段作为备用
- 优先使用 `repertory` 字段
- 如果没有 `repertory`，则使用原有的 `quantity` 或 `stockQuantity`
