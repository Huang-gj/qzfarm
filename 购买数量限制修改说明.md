# 购买数量限制修改说明

## 修改概述
将规格选择弹窗中的购买数量限制从写死的最大值2改为动态使用库存数量作为最大值，确保用户不能购买超过库存数量的商品。

### 1. 商品详情页面
**文件：** `Client/pages/goods/details/index.wxml`

#### 修改内容：
- **传递库存数量** (第113行)：

### 3. 规格选择弹窗逻辑
**文件：** `Client/pages/goods/details/components/goods-specs-popup/index.js`

#### 修改内容：

- **库存数量监听** (第27-37行)：
  ```javascript
  limitMaxCount: {
    type: Number,
    value: 999,
    observer(newVal) {
      console.log('[goods-specs-popup] 库存数量变化:', newVal);
      // 确保购买数量不超过库存
      const currentBuyNum = this.data.buyNum;
      if (currentBuyNum > newVal) {
        this.setData({
          buyNum: Math.max(1, newVal)
        });
      }
    }
  },
  ```

- **初始化数量限制** (第95-110行)：
  ```javascript
  // 确保初始购买数量不超过库存
  const { limitMaxCount } = this.properties;
  const currentBuyNum = this.data.buyNum;
  const validBuyNum = Math.min(currentBuyNum, limitMaxCount);
  
  console.log('[initData] 初始化购买数量:', {
    currentBuyNum: currentBuyNum,
    limitMaxCount: limitMaxCount,
    validBuyNum: validBuyNum
  });
  
  this.setData({
    specList,
    buyNum: validBuyNum
  });
  ```

- **数量变更处理** (第371-389行)：
  ```javascript
  handleBuyNumChange(e) {
    const { value } = e.detail;
    const { limitMaxCount, limitMinCount } = this.properties;
    
    // 确保数量在有效范围内
    const validValue = Math.max(limitMinCount, Math.min(value, limitMaxCount));
    
    console.log('[handleBuyNumChange] 数量变更:', {
      requestedValue: value,
      limitMinCount: limitMinCount,
      limitMaxCount: limitMaxCount,
      validValue: validValue
    });
    
    this.setData({
      buyNum: validValue,
    });
    
    // 触发数量变更事件
    this.triggerEvent('changeNum', {
      buyNum: validValue,
    });
  },
  ```

## 核心逻辑

### 数量限制规则：
1. **最小值**：`limitMinCount` (默认为1)
2. **最大值**：`limitMaxCount` (动态设置为库存数量)
3. **有效范围**：`[limitMinCount, limitMaxCount]`

### 数据流向：
1. **商品详情页面** → 传递 `stockQuantity` 作为 `limitMaxCount`
2. **规格选择弹窗** → 接收 `limitMaxCount` 并设置 `t-stepper` 的最大值
3. **数量变更** → 验证并限制在有效范围内

### 安全措施：
- **初始化检查**：组件初始化时确保购买数量不超过库存
- **实时验证**：数量变更时立即验证并调整
- **库存变化监听**：当库存数量变化时自动调整购买数量
- **详细日志**：添加调试信息便于问题排查

## 功能特点

1. **动态限制**：购买数量上限根据实际库存动态调整
2. **智能调整**：当库存不足时自动调整购买数量
3. **用户友好**：提供清晰的调试信息
4. **数据一致性**：确保前端显示与后端库存数据一致
